### Create Ticket Listing

List your tickets for sale on the marketplace.

**Location:** `/tickets/create` (requires authentication)

**Form Fields:**
- **Title**: Event name (required)
- **Description**: Detailed event information (required)
- **Category**: Concert, Sports, Theater, or Festival (required)
- **Event Date**: Date and time of the event (required)
- **Location**: Event venue or address (required)
- **Price**: Price per ticket in USD (required, positive number)
- **Quantity**: Number of tickets available (required, positive integer)

**Features:**
- Form validation for all fields
- Category selection dropdown
- Date picker for event date
- Real-time error messages
- Success notification on creation
- Automatic redirect to ticket details

**Business Logic:**
- Seller is automatically set to the authenticated user
- Available quantity initially equals total quantity
- Tickets appear immediately in marketplace
- Tickets can be edited or deleted by the seller

## Purchase Flow

Complete ticket purchase process.

### Purchase Tickets

**Steps:**
1. Browse tickets at `/tickets` or search for specific events
2. Click on a ticket to view details at `/tickets/[id]`
3. Select quantity (up to available stock)
4. Click "Purchase" button (requires authentication)
5. Confirm purchase details
6. Complete purchase

**Business Logic:**
- Must be authenticated to purchase
- Cannot purchase more than available quantity
- Cannot purchase your own tickets
- Price is calculated as: `quantity Ã— ticket price`
- Available quantity decreases after purchase
- Purchase record created with timestamp
- Seller and buyer are linked in database

**Validations:**
- Sufficient ticket availability
- Valid quantity (positive integer)
- User authentication
- Not purchasing own tickets

## Security Features

### Data Protection

- Password hashing with bcrypt
- Secure session management
- HTTPS enforcement in production
- HTTP-only cookies
- CSRF protection
- SQL injection prevention (via Prisma)

### Rate Limiting

API rate limits to prevent abuse:
- Authentication: 20 requests per 15 minutes
- Ticket creation: 10 requests per minute
- Purchases: 5 requests per minute
- Browse tickets: 30 requests per minute

**Learn more:** [Rate Limiting Guide](/docs/rate-limiting)

### Access Control

User data isolation:
- Users can only edit/delete their own tickets
- Purchase history is private
- Dashboard shows only user's data
- Protected routes require authentication
- Authorization checks on all mutations
